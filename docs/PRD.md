# 📄 [Vibe-Code Auditor] 요구사항 정의서 (PRD.md)

## 프로젝트 개요
**바이브코딩으로 개발한 프로젝트를 AI + 정적 분석 도구로 자동 점검하고, 배포/자가사용 관점에서 개선 방향을 제시하는 CLI 기반 코드 감사 도구**

## 핵심 타겟
- **페르소나**: 바이브코딩을 활용해 프로젝트를 개발했으나, 코드 품질/보안/최적화 상태를 점검하고 싶은 비개발자 또는 초급 개발자
- **사용 빈도**: 프로젝트 완성 후 배포 전, 또는 주기적인 유지보수 시점

## 주요 기능 (MVP Scope)

### 필수 기능 1: 로컬 프로젝트 경로 입력 및 분석 모드 선택
- 사용자가 로컬 폴더 경로 입력
- 분석 관점 선택:
  - **배포 관점**: 보안, 성능, 확장성, CI/CD 검증
  - **자가 사용 관점**: 가독성, 유지보수성, 코드 중복 제거

### 필수 기능 2: 범용 코드 분석 (AI + 정적 분석 도구 조합)
- **AI 분석** (Claude Code API 활용):
  - 주관적 코드 리뷰 (오타, 오래된 주석, 오해의 소지가 있는 함수명 등)
  - 아키텍처 개선 제안
- **정적 분석 도구** (언어별 자동 감지 후 실행):
  - Python → Pylint
  - JavaScript/TypeScript → ESLint
  - 범용 보안 스캔 → Semgrep
  - 코드 중복 감지 → jscpd
  - 종합 품질 분석 → SonarQube (선택적)

### 필수 기능 3: 터미널 CLI 텍스트 리포트 출력
- 분석 결과를 색상 구분된 텍스트로 출력:
  - 🔴 **Critical**: 보안 취약점, 치명적 버그
  - 🟡 **Warning**: 코드 중복, 복잡도 초과, 성능 이슈
  - 🟢 **Info**: 개선 제안, 리팩토링 방향
- 선택한 관점(배포/자가사용)에 따라 우선순위 필터링

## 그 외 기능은 추후 고도화로 미룸
- GitHub URL 자동 클론 기능
- HTML/PDF 리포트 생성
- 웹 대시보드 UI
- 자동 수정 기능 (AI가 코드 직접 패치)

## 사용자 흐름 (User Flow)
1. **접속**: 사용자가 터미널에서 `vibe-auditor` 명령 실행
2. **핵심 기능 사용**:
   - 로컬 프로젝트 경로 입력
   - 분석 관점 선택 (배포 vs 자가사용)
   - 자동 분석 시작 (AI + 정적 분석 도구)
3. **완료**: 컬러 텍스트 리포트 출력 → 개선 방향 확인
